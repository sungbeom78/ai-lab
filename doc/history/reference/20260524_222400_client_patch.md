# BomTS Technical Reference - OpenClaw Client-Side File Patch Specification

본 문서는 분산 네트워크 환경에서 AI 모델이 출력한 Proposed Files를 클라이언트 측 VS Code 에디터가 직접 실시간으로 안전하게 가로채어 실제 파일 시스템에 패치(Apply)하기 위한 파싱 알고리즘 및 네이티브 워크스페이스 API(`vscode.workspace.fs.writeFile`) 호출 연동 명세서입니다.

---

## 1. 클라이언트 마크다운 파서 및 정규식 설계

노트북 VS Code 웹뷰 패널 자바스크립트에 내장된 Proposed Files 추출용 정규식(Regex) 알고리즘입니다.

* **정규식 패턴**: `/\`\`\`[a-zA-Z0-9_\\-#\\+]*\\n([\\s\\S]*?)\`\`\`/g`
  * 백틱 이스케이프와 달러 이스케이프(\` 및 \${ })를 처리하기 위해, 백그라운드 문자열 내에서는 더블 역슬래시(`\\`)와 이스케이프 백틱(`\``)으로 조립되어 린트 오류 없이 안전하게 파싱을 전담합니다.
* **알고리즘 흐름**:
  1. 수신한 `msg.answer` 문자열에서 코드 블록들을 순회 탐색합니다.
  2. 코드 블록 내용 중 상위 3줄 이내에 `### FILE: <절대경로>` 지시 태그가 존재하는지 체크합니다.
  3. 지시 태그가 감지되면, 주석 태그 라인을 지운 알맹이 소스 코드 본문을 정밀 추출하여 Extension 호스트로 송출합니다.

---

## 2. VS Code Extension 네이티브 쓰기 핵심 API

수집된 proposed files를 노트북이 실제로 제어하고 있는 워크스페이스 상의 연동 파일에 실제로 물리적으로 기입하고 저장하는 TypeScript 구현체 구조입니다.

```typescript
case 'applyFilesClient': {
    const files: any[] = data.files || [];
    const workspaceFolders = vscode.workspace.workspaceFolders;
    const workspaceRoot = workspaceFolders ? workspaceFolders[0].uri.fsPath : '';
    
    for (const f of files) {
        let targetPath = f.path;
        if (!path.isAbsolute(targetPath)) {
            targetPath = path.join(workspaceRoot, targetPath);
        }
        
        try {
            const targetUri = vscode.Uri.file(targetPath);
            const contentBuffer = Buffer.from(f.content, 'utf8');
            
            // 1) VS Code Workspace File System API를 활용하여 실제 물리적 저장소에 데이터 주입
            await vscode.workspace.fs.writeFile(targetUri, contentBuffer);
            
            // 2) 소스코드 쓰기가 완료되는 즉시 사용자의 화면에 변경 사항이 담긴 탭을 강제 기동하여 시각화!
            const doc = await vscode.workspace.openTextDocument(targetUri);
            await vscode.window.showTextDocument(doc, { preview: false });
            
        } catch (err: any) {
            // 에러 핸들링
        }
    }
    break;
}
```

## 3. 분산망 격리 탈출 효과 (Architecture Verification)

기존에는 AI 모델의 출력이 Desktop 로컬의 엉뚱한 임시 저장소로 격리되어 소용이 없었으나, 이 아키텍처적 개선으로 노트북 클라이언트단이 직접 API를 기동하게 됨으로써 노트북-NUC간에 마운트되거나 로컬로 열려있는 실제 사용자의 활성 에디터 파일을 정확하게 추적하여 실시간 반영이 완벽히 동작하게 됩니다.
