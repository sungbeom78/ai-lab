# Ahnda 개인 작업실 로그 - 2026-05-24 (File Picker)

기획 파일 경로를 기입할 때 겪던 오타율과 경로 탐색 번거로움을 원천 방어하기 위해 오늘 오픈클로 웹뷰와 **VS Code 네이티브 파일 탐색 팝업(showOpenDialog)**을 성공적으로 바인딩했다. 

## 1. 깨달음과 개선 흔적 (Reflection)
경로 입력 시 오타가 발생하면 백엔드에서 기획서 읽기를 실패하여 일반론적인 답변만 나오는 부작용이 크다.
이를 보완하기 위해 웹뷰 인풋 옆에 돋보기 버튼을 달고, 클릭 시 VS Code 자체 네이티브 탐색기를 띄워 마크다운 문서를 마우스로 툭 선택하면 경로가 슥 들어가도록 아키텍처를 유려하게 연동했다.
또한, 사용자가 지정한 기획 지침 파일에 대해 AI 모델이 **"절대적으로 복종하고 1순위로 엄수하여 코딩해야 할 절대 명령(Absolute Rule)"**임을 프롬프트 단에 강력하게 규정해 두었다.
이제 기획 의도를 한 치의 오차도 없이 온전하고 정확하게 파일 시스템 변경에 투영할 수 있게 되었다!
