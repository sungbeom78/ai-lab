# Ahnda 개인 작업실 로그 - 2026-05-24

오늘의 작업 목표는 로컬 AI 개발 시 가장 큰 걸림돌이었던 **"코드 개발 대신 요약만 하던 현상"**을 수정하고, 동시에 OpenClaw의 지침 전달력과 활용도를 끌어올리기 위한 모드(Mode) 다중 선택 기능을 구현하는 것이었다.

## 1. 오늘의 깨달음과 생각 (Reflection)
AI 모델(특히 로컬의 Qwen이나 Gemma 같은 모델)들은 시스템 프롬프트(System Prompt) 지침의 어조가 조금이라도 모호하거나 일반론적으로 선언되어 있으면, 바로 코드를 짜기보다 요약문이나 단계를 나열하는 편한 방식으로 우회해 버린다.
이를 막기 위해, 백엔드 단의 프롬프트 빌더에 **"말만 하지 말고 무조건 Proposed Files에 복사해서 바로 쓸 수 있는 전체 소스코드를 중략 없이 온전하게 출력하라"**는 명확하고 강력한 제약(요약 금지 법칙)을 주입해 주는 해결 방법을 적용했다.
또한, 사용자가 `inspect`와 `develop`을 동시에 선택하여 "구조 분석과 실제 코드 수정 개발"을 동시에 한 번에 AI에게 온전히 지시할 수 있도록 UI를 다중 체크박스로 리팩토링했다.

---

## 2. 수행한 작업 (Actions Taken)
* **OpenClaw Extension Webview UI 리팩토링**:
  * `<select>` 엘리먼트를 여러 개 선택할 수 있는 다중 체크박스 테이블 형태로 전면 개편했다.
  * 체크된 여러 모드들을 쉼표로 연결하여 API 서버로 한 번에 날려주도록 프론트 스크립트를 작성했다.
* **백엔드 프롬프트 조립 로직 대폭 강화**:
  * 쉼표로 넘어온 다중 모드를 분석해 각 모드에 따른 고정 관념 지침(Action Directives)을 시스템 프롬프트의 핵심 최우선 규칙으로 동적 배치했다.
  * 특히 `develop` 모드에서 코드를 온전히 작성하게 하는 초강력 주입 명령을 선언했다.
* **컴파일 및 패키징 빌드 성공**:
  * `npm run compile` 및 `vsce package`를 통해 `openclaw-vscode-0.0.1.vsix`를 성공적으로 최신화 빌드했다.
