# 기술 블로그 초안: 로컬 AI 콘솔의 독립 VENV 구축과 포트 포워딩 설계

로컬 환경에 나만의 AI 콘솔(Ollama Web Console)을 구축할 때 마주치는 2가지 흔한 문제가 있습니다. 
첫째, 파이썬 패키지 충돌. 둘째, 권한 문제와 네트워크 포워딩입니다.

이번 작업에서는 아래 방법으로 문제를 해결했습니다.

## 1. VENV 환경 분리
리눅스의 기본 Python 환경을 오염시키지 않기 위해 `.venv`를 구축합니다. `setup_local_ai_console_venv.sh`를 통해 원클릭으로 패키지를 격리하고, 구동 스크립트가 이를 자동으로 감지하여 실행하게 만들었습니다.

## 2. 안전한 비특권 포트(11004) 사용
1000번대 이하 포트는 `sudo` 권한을 요구하기 때문에 자동화 및 일반 사용자 실행에 큰 걸림돌이 됩니다. 이를 11004 포트로 옮겨 권한 이슈를 원천 차단했습니다.

## 3. WSL2 외부 개방 (Portproxy)
로컬에 구축한 AI 콘솔을 같은 망 내의 다른 기기(노트북 등)에서 쓰기 위해 Windows의 `netsh interface portproxy`와 방화벽 인바운드 허용 규칙을 스크립트화 했습니다. (이를 통해 보안을 통제하면서도 유연한 접근이 가능해집니다.)
